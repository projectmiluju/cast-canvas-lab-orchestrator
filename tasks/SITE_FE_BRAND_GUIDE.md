# Site-First Brand Guide

이 문서는 `cast-canvas-lab-site` 선행 디자인 리뉴얼과 `cast-canvas-lab-fe` 후속 브랜드 정렬 작업의 공통 기준이다.

- ORCH 에픽: `projectmiluju/cast-canvas-lab-orchestrator#8`
- SITE 선행 이슈: `projectmiluju/cast-canvas-lab-site#4`
- FE 후속 이슈: `projectmiluju/cast-canvas-lab-fe#22`

## 목적

- `site`에서 브랜드 인상을 먼저 확정한다.
- `frontend`는 같은 브랜드를 느끼되, 앱 사용성을 해치지 않는 밀도로 재해석한다.
- Google Stitch MCP와 Claude를 활용한 초안 생성 및 수정 작업이 흔들리지 않도록 기준을 고정한다.

## 작업 순서

1. `site`에서 Google Stitch MCP로 초안을 만든다.
2. Claude가 `site` 초안을 실제 제품 톤에 맞게 구조와 비주얼을 수정한다.
3. `site` 결과에서 공통 브랜드 요소를 추출한다.
4. `frontend`에 공통 요소를 절제된 형태로 반영한다.
5. 두 레포의 최종 결과가 같은 제품 경험인지 교차 검토한다.

## 도구 역할 분담

- Google Stitch MCP: 초기 시안 생성, 섹션 조합, 시각 방향 탐색
- Claude: 정보 구조 정리, 과장 제거, 실제 레포 코드 수정, 세부 톤 정렬

## 브랜드 키워드

- Focused: 산만하지 않고 핵심 정보가 또렷해야 한다.
- Spatial: 평면 문서 도구가 아니라 공간형 워크스페이스라는 인상이 보여야 한다.
- Editorial: 문서 친화적이고 읽기 쉬운 리듬을 가져야 한다.
- Precise: 과장된 스타트업 랜딩 감성보다 정제된 제품 감각이 우선이다.
- Quietly Confident: 과한 장식 없이도 완성도가 느껴져야 한다.

## 공통 시각 원칙

### 1. Color

- 강한 포인트 컬러는 1개 축만 사용한다.
- 배경은 눈부신 순백 대신 약간 톤이 깔린 중립 배경을 우선한다.
- 인터랙션 컬러는 CTA와 상태 강조에만 집중한다.
- `site`와 `frontend`는 같은 계열의 주요 색을 쓰되, `frontend`는 대비와 피로도를 더 엄격히 관리한다.

### 2. Typography

- 문서형 제품답게 읽기 좋은 타이포 리듬을 우선한다.
- 헤드라인은 선명하고 구조적이어야 하며, 본문은 설명 밀도가 높아져도 버텨야 한다.
- `site`는 표현력이 조금 더 커도 되지만, `frontend`는 스캔 속도와 정보 밀도가 우선이다.

### 3. Shape And Surface

- 모서리, 보더, 표면 톤은 둥글더라도 지나치게 말랑하면 안 된다.
- 카드와 패널은 떠 있는 느낌보다 정리된 작업면의 인상을 준다.
- 그림자는 존재감보다 계층 구분 용도로만 사용한다.

### 4. Layout Rhythm

- 공간감을 보여주되 섹션 간 호흡은 단정해야 한다.
- `site`는 넓은 여백과 시각적 리듬을 활용할 수 있다.
- `frontend`는 작업 효율을 해치지 않는 범위에서만 여백을 확장한다.

### 5. Motion

- 의미 있는 전환만 허용한다.
- `site`는 진입 애니메이션과 섹션 reveal을 제한적으로 사용할 수 있다.
- `frontend`는 상태 변화와 집중 전환 위주로만 움직인다.

## 레포별 적용 원칙

## `site`

- 브랜드 첫인상을 결정하는 기준 저장소다.
- hero, 제품 소개, 핵심 가치, 앱 진입 CTA에서 톤을 확정한다.
- 더 과감한 레이아웃과 비주얼 시도를 허용한다.
- 단, 실제 제품 상태보다 앞서간 약속이나 과장된 마케팅 문구는 금지한다.

## `frontend`

- 브랜드를 사용하는 공간이다.
- 색, 타이포, 컴포넌트 형태 언어는 `site`와 맞추되 정보 밀도는 더 높게 유지한다.
- 로그인/회원가입, 워크스페이스 진입, 공통 버튼/입력/패널부터 우선 정렬한다.
- 캔버스 영역은 브랜딩보다 작업 효율이 우선이다.

## 공통 컴포넌트 기준

- Button: `site`와 `frontend` 모두 같은 성격의 primary/secondary 계층을 유지한다.
- Card/Panel: 동일한 표면 언어를 공유하되, `frontend`는 더 얇고 명확한 계층을 사용한다.
- Input: 장식보다 가독성과 포커스 상태를 우선한다.
- Navigation/CTA: 주요 행동은 같은 톤으로 강조하되, 앱 내부에서는 빈도를 절제한다.

## 금지사항

- `site`와 `frontend`가 각자 다른 제품처럼 보이게 만드는 독립적 스타일 실험
- 과한 글래스모피즘, 과도한 블러, 과장된 3D 표현
- 다수의 포인트 컬러를 섞어 브랜드 축을 흐리는 구성
- `frontend`의 가독성이나 작업 효율을 희생하는 시각 효과
- 미구현 기능을 이미 제공하는 것처럼 보이게 만드는 카피/그래픽
